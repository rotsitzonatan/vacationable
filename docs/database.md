# Database

## Database
PostgreSQL

## Main Entities
- User
- Hotel
- Room
- Booking
- Review
- Image
- Amenity

## Relationships
- One User can own multiple Hotels.
- One Hotel contains multiple Rooms.
- One User can create multiple Bookings.
- One Room can have multiple Bookings over time.
- One Booking may have one Review.
- Hotels and Rooms can have multiple Images.
- Hotels can have multiple Amenities.

## Notes
The database schema will evolve during development as new business requirements are introduced.
